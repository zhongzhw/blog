led

led模块驱动

配置文件 CONFIG_ARCH_LEDS=y 表明nsh控制led，用户不能操作

led模块有两种驱动模式，板载函数驱动方式及设备注册驱动方式

板载函数驱动方式

函数定义

include/nuttx/board.h 中含有led使用函数申明

/* led初始化 */
#ifdef CONFIG_ARCH_HAVE_LEDS
uint32_t board_userled_initialize(void);
#endif

/* 设置相应的led灯状态 */
#ifdef CONFIG_ARCH_HAVE_LEDS
void board_userled(int led, bool ledon);
#endif

/* 设置所有的led灯状态 */
#ifdef CONFIG_ARCH_HAVE_LEDS
void board_userled_all(uint32_t ledset);
#endif

可在上述头文件中自定义led操作函数

函数体实现

boards/<arch>/<chip>/<board>/src/initapp.c文件为对应控制板的启动文件（上电后会自动执行该文件） boards/<arch>/<chip>/<board>/src/userleds.c中包含上述函数申明的具体实现

使用

在自定义app程序中包含头文件

#include <arch/board/board.h>

可使用1中定义的函数操作led

设备注册驱动方式

在开发板启动文件 boards/<arch>/<chip>/<board>/src/initapp.c 或 boards/<arch>/<chip>/<board>/src/bringup.c 中添加设备注册

#ifndef CONFIG_ARCH_LEDS
  userled_lower_initialize("/dev/led");
#endif

可在自定义app程序中直接使用
